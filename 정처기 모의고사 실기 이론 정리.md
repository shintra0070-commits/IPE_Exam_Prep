### DIP(의존성 역전 원칙)

| 내용 |
| --- |
| 고수준 모듈이 저수준 모듈에 직접 의존하지 않고, 둘 다 추상화(인터페이스·추상 클래스)에 의존해야 한다는 원칙. |

### 모듈 호출 지표

| 구분 | 내용 |
| --- | --- |
| 특정 모듈을 호출하는 상위 모듈의 수 | Fan-In |
| 특정 모듈이 호출하는 하위 모듈의 수 | Fan-Out |

### 백업 유형

| 구분 | 내용 |
| --- | --- |
| 풀 백업 | 전체 데이터를 매번 백업하여 복구가 단순하고 빠르지만 백업 시간과 저장 공간을 많이 사용한다. |
| 중분 백업 | 마지막 백업 이후 변경된 부분만 백업하여 시간·공간을 절약하지만 복구 시 여러 백업 파일이 필요하다. |

| 내용 |
| --- |

### 전문 인덱스(Full-text Index)

| 내용 |
| --- |

| 내용 |
| --- |

### FIN(Finish)

| 내용 |
| --- |
| TCP 통신에서 연결 종료 과정(4-way handshake) 중 능동적으로 연결을 끊으려는 측에서 가장 먼저 전송하는 제어 플래그 |

### 디지털 트윈

| 내용 |
| --- |
| 현실의 물리적 사물이나 시스템을 가상 공간에 동일하게 구현하고, 센서 등에서 수집한 실시간 데이터로 동기화하여 모의실험·분석·예측에 활용하는 기술 |

### 2단계 로킹 프로토콜(2PL)

| 내용 |
| --- |
| 동시성 제어 기법 중, 트랜잭션이 잠금(Lock)을 획득하기만 하는 확장 단계와 잠금을 해제하기만 하는 수축 단계로 나뉘며, 한 번 잠금을 해제하면 새로운 잠금을 얻을 수 없도록 하여 직렬 가능성을 보장하는 프로토콜 |

### OCSP

| 내용 |
| --- |
| PKI(Public Key Infrastructure) 환경에서 디지털 인증서의 유효성을 검증할 때, 인증서가 폐기되었는지 실시간으로 확인하기 위해 사용하는 프로토콜 |

### 제로 트러스트

| 내용 |
| --- |
| 절대 신뢰하지 않고 항상 검증한다(Never Trust, Always Verify)'를 핵심 원칙으로, 네트워크 내부·외부를 구분하지 않고 모든 접근 요청을 검증하는 보안 모델 |

### 엣지 컴퓨팅

| 내용 |
| --- |
| 중앙 클라우드나 데이터센터가 아니라 데이터가 발생하는 단말이나 현장 가까이에서 데이터를 처리하여, 응답 지연(latency)과 네트워크 부하를 줄이는 분산 컴퓨팅 방식 |

### SYN Flooding

| 내용 |
| --- |
| - TCP는 연결을 수립할 때 클라이언트가 SYN 패킷을 보내고, 서버가 SYN+ACK로 응답한 뒤, 클라이언트가 다시 ACK를 보내는 3-Way Handshake 과정을 거친다.<br>- 공격자는 출발지 IP를 위조한 다수의 SYN 패킷을 서버로 전송한 뒤 마지막 ACK를 고의로 보내지 않아, 서버가 응답을 기다리는 '반열림(half-open)' 연결을 계속 유지하게 만든다.<br>- 이로 인해 서버의 연결 대기 큐(backlog queue)가 가득 차 정상 사용자의 접속 요청을 더 이상 처리하지 못하게 되는 서비스 거부(DoS) 공격이다. |

## 정처기 실기 모의고사 31회

### 설계 원칙

| 구분 | 내용 |
| --- | --- |
| LSP(리스코프 치환 원칙) | 상위 타입의 객체를 하위 타입의 객체로 치환해도 프로그램이 정상 동작해야 한다. |
| DIP(의존성 역전 원칙) | 고수준 모듈이 저수준 모듈에 의존하지 않고, 둘 다 추상화에 의존해야 한다. |

### 나선형 모델

| 내용 |
| --- |
| 초기 요구사항이 불명확하고 위험 요소가 큰 프로젝트에서, 각 반복 주기마다 계획 수립 → 위험 분석 → 개발/검증 → 고객 평가를 수행하며 점진적으로 시스템을 개발하는 소프트웨어 개발 생명주기 모델 |

### 비트맵 인덱스

| 내용 |
| --- |
| OLTP 환경에서 수십만 개의 고유 값을 가진 주문번호 컬럼과 달리, OLAP 환경에서 '상품분류' 컬럼처럼 카디널리티가 극히 낮은(10개 미만) 차원 데이터에 최적화된 인덱스이고 각 고유 값마다 별도의 비트 벡터를 유지하며, 복합 조건 검색 시 비트 연산(AND, OR, NOT)을 통해 매우 빠른 집계 처리를 제공한다. |
