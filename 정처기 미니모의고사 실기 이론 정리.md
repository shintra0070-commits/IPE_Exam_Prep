## 미니 모의고사 2회

- **리펙토링**: 복잡한 코드를 단순화하고 가독성을 높여 유지보수성과 생산성, 품질을 향상시키며 더 유연한 시스템을 만드는 것이 목적
- **T1**: `SELECT COUNT(*) FROM orders WHERE status = 'PENDING'`
- **T2**: `INSERT INTO orders VALUES (101, 'PENDING')`
- **T1**: `SELECT COUNT(*) FROM orders WHERE status = 'PENDING' (재실행)`
  - **정답**: 팬텀리드(트랜잭션 이상 현상명칭)
- **(SJF) 스케줄링**: Ready Queue 에 있는 프로세스 중 CPU 처리 시간이 짧은 순서대로 CPU 를 할당하는 비선점형 방식이며, 늦게 도착하더라도 CPU 처리 시간이 앞에 대기 중인 프로세스보다 짧으면 먼저 CPU 를 할당받을 수 있다.
- **(RR) 스케줄링**: 프로세스가 도착한 순서대로 디스패치하지만 정해진 시간 할당량(time quantum)에 의해 실행을 제한한다. 시간 할당량을 매 프로세스에 주고 할당된 시간 안에 완료되지 못한 프로세스는 준비 큐의 맨 뒤에 배치되도록 하여 CPU 를 독점하지 않고 공평하게 이용할 수 있게 한다.

## 미니 모의고사 5회

- **Hub-and-Spoke**: 기업 내 애플리케이션 통합(EAI) 구축 유형 중 중앙의 허브를 두고 데이터를 전송하는 방식
- 요구조건 분석 → 개념적 설계(E-R 모델 등) → 논리적 설계(테이블·정규화) → 물리적 설계(저장 구조·인덱스) → 구현(DDL 생성)
- **역상 저항성**: 블록체인에서 사용되는 SHA-256 해시 함수의 특성 중, 주어진 해시값에 대해 그 해시값을 생성한 원본 입력값을 찾는 것이 계산적으로 불가능해야 한다는 특성
- TCP는 연결을 수립할 때 클라이언트가 SYN 패킷을 보내고, 서버가 SYN+ACK로 응답한 뒤, 클라이언트가 다시 ACK를 보내는 3-Way Handshake 과정을 거친다.
- 공격자는 출발지 IP를 위조한 다수의 SYN 패킷을 서버로 전송한 뒤 마지막 ACK를 고의로 보내지 않아, 서버가 응답을 기다리는 '반열림(half-open)' 연결을 계속 유지하게 만든다.
- 이로 인해 서버의 연결 대기 큐(backlog queue)가 가득 차 정상 사용자의 접속 요청을 더 이상 처리하지 못하게 되는 서비스 거부(DoS) 공격이다. -> SYN Flooding
