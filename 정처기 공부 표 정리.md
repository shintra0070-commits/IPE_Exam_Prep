## 2. 코딩 핵심 정리

| 언어 | 핵심 항목 | 반드시 기억할 내용 | 공부 포인트 |
|---|---|---|---|
| C | `printf` 서식지정자 | `%d` 정수, `%c` 문자, `%s` 문자열, `%f` 실수, `%x` 16진수, `%o` 8진수 | 출력 형식과 자료형을 같이 확인 |
| C | 정수 나눗셈 | C/Java에서 `5 / 2 = 2`, Python에서 `5 / 2 = 2.5`, Python `5 // 2 = 2` | 언어별 나눗셈 결과 구분 |
| C | 포인터 | `&a`는 주소, `p`는 주소 저장, `*p`는 가리키는 값 | 주소와 값을 표로 나눠 추적 |
| C | 포인터 연산 | `*(p+1)`은 다음 칸의 값, `*p+1`은 현재 값에 1 더함 | 괄호 위치를 가장 먼저 확인 |
| C | 배열 | 배열 이름은 첫 번째 원소의 주소처럼 동작 | 인덱스와 주소 이동을 함께 보기 |
| C | 구조체 포인터 | `p->x`는 `(*p).x`와 같음 | `.`와 `->` 사용 조건 구분 |
| C | 심화 빈출 | 문자열 포인터, 구조체 포인터, 이중 포인터, 2차원 배열, 연결 리스트, 큐, 스택, `static`, `malloc`, `#define`, `ctype.h` | 포인터/배열/구조체는 그림으로 풀기 |
| Java | 객체지향 기본 | 클래스, 객체, 생성자, `static`, 상속, 인터페이스, 다형성 | 참조 타입과 실제 객체 타입을 구분 |
| Java | 오버라이딩 | 메서드는 실제 객체 기준으로 실행 | 실행되는 메서드가 어느 클래스 것인지 표시 |
| Java | 필드 접근 | 필드는 동적 바인딩되지 않고 참조 타입 기준으로 접근 | 메서드와 필드의 기준 차이 암기 |
| Java | 오버로딩/오버라이딩 | 오버로딩은 같은 이름 다른 매개변수, 오버라이딩은 부모 메서드 재정의 | 문제에서 매개변수와 상속 관계 확인 |
| Java | `String` 비교 | `==`는 주소 비교, `.equals()`는 내용 비교 | 문자열 비교 문제에서 최우선 확인 |
| Java | 길이 | 배열은 `배열.length`, 문자열은 `문자열.length()` | 괄호 유무 주의 |
| Java | 자주 헷갈림 | String Pool, 기본형 vs 참조형 변수 | 값 비교인지 주소 비교인지 판단 |
| Python | 자료형 추적 | 리스트, 문자열, 딕셔너리, 집합, 튜플 | 인덱스와 값 변화를 직접 쓰기 |
| Python | 슬라이싱 | `a[1:4]`는 1번부터 3번까지, `a[::-1]`은 역순, `a[::2]`는 2칸 간격 | 시작, 끝, 간격을 분리해서 계산 |
| Python | 문자열 메서드 | `split`, `strip`, `replace`, `find`, `startswith`, `upper`, `lower` | 반환값과 원본 변경 여부 확인 |
| Python | 딕셔너리 | `keys`, `values`, `items` | 반복문에서 무엇이 나오는지 확인 |
| Python | 집합 | `add`, `update`, `remove`, `discard` | `remove`와 `discard` 차이 주의 |
| Python | 튜플 | 튜플은 불변, `count`, `index` 확인 | 값 변경 가능 여부 판단 |
| Python | 기타 | 컴프리헨션, 언패킹, `enumerate`, 함수, 람다, `map`, 클래스 | 출력 추적형 문제로 연습 |

## 9. 최종 공부 우선순위

| 우선순위 | 공부 항목 | 이유 |
|---|---|---|

## 10. 시험 직전 체크법

| 체크 방법 | 확인 기준 |
|---|---|
| 목차만 보고 뜻 말하기 | 항목명을 보고 3초 안에 정의가 떠오르는지 확인 |
| 코딩 문제 손추적 | 변수 변화, 주소 변화, 배열 값을 직접 표로 작성 |
| 암기 파트 한 줄 정의 | 긴 설명보다 정확한 키워드 중심으로 반복 |
| 헷갈리는 항목 표시 | `==`/`.equals()`, `remove`/`discard`, 응집도/결합도 순서처럼 혼동 포인트 따로 암기 |
