### JSON

| 내용 |
| --- |
| 속성-값(Attribute-value Pair)으로 이루어진 데이터 오브젝트를 전달하기 위해 사용하는 개방형 표준 포맷이다. AJAX(Asynchronous JavaScript and XML)에서 많이 사용되고 XML(eXtensible Markup Language)을 대체하는 주요 데이터 포맷이다. 언어 독립형 데이터 포맷으로 다양한 프로그래밍 언어에서 사용하고 있는 기술이다 |

### 랜드 어택(Land Attack)

| 내용 |
| --- |
| 공격자가 패킷의 출발지 주소나 포트를 임의로 변경해 출발지와 목적지 주소를 동일하게 함으로써 공격 대상 컴퓨터의 실행속도를 느리게 하거나 동작을 마비시켜 서비스 거부 상태에 빠지도록 하는 공격 방법 |

### MD5

| 내용 |
| --- |
| RFC 1321로 지정되어 있으며, 주로 프로그램이나 파일이 원본 그대로 인지를 확인하는 무결성 검사 등에 사용된다. 1991년 로널드 라이베스트가 예전에 쓰이던 MD4를 대체하기 위해 고안된 128비트 암호화 해시 함수 |

### OSI 계층

| 구분 | 내용 |
| --- | --- |
| 물리 계층(Physical) | 0과 1의 비트(Bit)를 전기적·기계적 신호로 전송하는 계층 |
| 데이터링크 계층(Data Link) | 프레임(Frame) 단위로 데이터를 전송하며 오류 검출 및 MAC 주소를 이용해 통신하는 계층 |
| 네트워크 계층(Network) | **패킷(Packet)을 목적지까지 전달하기 위한 경로를 설정(라우팅)**하는 계층 |
| 전송 계층(Transport) | 세그먼트(Segment) 단위로 데이터를 신뢰성 있게 전송하고 오류 및 흐름을 제어하는 계층 |
| 세션 계층(Session) | 통신 세션을 설정·유지·종료하는 계층 |
| 표현 계층(Presentation) | 데이터 형식 변환, 암호화/복호화, 압축/해제를 수행하는 계층 |
| 응용 계층(Application) | 사용자에게 네트워크 서비스를 제공하는 계층 (웹, 이메일 등) |

### 에플리케이션의 성능 측정 지표

| 구분 | 내용 |
| --- | --- |
| 처리량 | 애플리케이션이 주어진 시간에 처리할 수 있는 트랜잭션의 수 |
| 응답 시간 | 사용자 입력이 끝난 후, 애플리케이션의 응답 출력이 개시될 때까지의 시간 |
| 경과 시간 | 애플리케이션에 사용자가 요구를 입력한 시점부터 트랜잭션 처리 후 그 결과의 출력이 완료될 때까지 걸리는 시간 |
| 자원 사용률 | 애플리케이션이 트랜잭션을 처리하는 동안 사용하는 CPU 사용량, 메모리 사용량, 네트워크 사용량 |

### AJAX

| 내용 |
| --- |
| 브라우저가 가지고 있는 XMLHttpRequest 객체를 이용해서 전체 페이지를 새로 고치지 않고도 페이지의 일부분만을 위한 데이터를 로드하는 기법이며, 하이퍼텍스트 표기 언어(HTML)만으로 어려운 다양한 작업을 웹 페이지에서 구현해 이용자가 웹 페이지와 자유롭게 상호작용할 수 있도록 하는 기술명 |

### 애자일 방법론

| 내용 |
| --- |
| 절차보다는 사람이 중심이 되어 변화에 유연하고 신속하게 적응하면서 효율적으로 시스템을 개발할 수 있는 신속 적응적 경량 개발방법론으로, 개발 기간이 짧고 신속하며, 워터폴에 대비되는 방법론으로 최근 회사에서 각광받는 방법론 |

### Rollback

| 내용 |
| --- |
| 랜잭션 수행 중 오류가 발생하거나 사용자가 작업을 취소할 때, 해당 트랜잭션이 변경한 내용을 취소하고 트랜잭션 시작 시점 또는 지정된 SAVEPOINT 등 이전의 일관된 상태로 데이터베이스를 되돌리는 트랜잭션 제어 명령(TCL)이다. |

### IPSec(Internet Protocol Security

| 내용 |
| --- |
| 무결성과 인증을 보장하는 인증헤더(AH)와 기밀성을 보장하는 암호화(ESP)를 이용한 프로토콜로 네트워크 계층(Network Layer)인 인터넷 프로토콜(IP)에 보안성을 제공해주는 표준화된 기술 |

| --- |

### 생성(Creational) 패턴

| 구분 | 내용 |
| --- | --- |
| 싱글톤(Singleton) 패턴 | 클래스의 인스턴스를 하나만 생성하고 어디서든 동일한 객체를 사용하는 패턴 |
| 팩토리 메서드(Factory Method) 패턴 | 객체 생성을 서브클래스에 위임하여 객체를 생성하는 패턴 |
| 추상 팩토리(Abstract Factory) 패턴 | 관련된 객체들을 하나의 팩토리에서 생성하는 패턴 |
| 빌더(Builder) 패턴 | 객체를 단계적으로 생성하여 복잡한 객체를 만드는 패턴 |
| 프로토타입(Prototype) 패턴 | 기존 객체를 복제(Clone) 하여 새로운 객체를 생성하는 패턴 |

### 구조(Structural) 패턴

| 구분 | 내용 |
| --- | --- |
| 어댑터(Adapter) 패턴 | 호환되지 않는 인터페이스를 연결하여 함께 사용할 수 있도록 하는 패턴 |
| 브리지(Bridge) 패턴 | 구현과 추상을 분리하여 독립적으로 확장할 수 있게 하는 패턴 |
| 컴포지트(Composite) 패턴 | 트리 구조로 객체를 구성하여 부분과 전체를 동일하게 처리하는 패턴 |
| 데코레이터(Decorator) 패턴 | 기존 객체를 수정하지 않고 기능을 동적으로 추가하는 패턴 |
| 퍼사드(Facade) 패턴 | 복잡한 시스템을 하나의 간단한 인터페이스로 제공하는 패턴 |
| 프록시(Proxy) 패턴 | 대리 객체를 통해 실제 객체에 접근을 제어하는 패턴 |

### 행위(Behavioral) 패턴

| 구분 | 내용 |
| --- | --- |
| 옵서버(Observer) 패턴 | 한 객체의 상태가 변경되면 관련 객체들에게 자동으로 변경 내용을 전달하는 일대다 의존성 패턴 |
| 전략(Strategy) 패턴 | 알고리즘을 캡슐화하여 상황에 따라 교체할 수 있는 패턴 |
| 상태(State) 패턴 | 객체의 상태에 따라 동작이 변경되도록 하는 패턴 |
| 커맨드(Command) 패턴 | 요청을 객체로 캡슐화하여 실행, 취소 등을 처리하는 패턴 |
| 템플릿 메서드(Template Method) 패턴 | 처리 순서는 상위 클래스에서 정의하고 세부 구현은 하위 클래스에서 수행하는 패턴 |
| 중재자(Mediator) 패턴 | 객체 간의 직접적인 통신을 중재 객체가 담당하여 결합도를 낮추는 패턴 |
| 반복자(Iterator) 패턴 | 컬렉션의 내부 구조를 노출하지 않고 순차적으로 접근하는 패턴 |
| 메멘토(Memento) 패턴 | 객체의 이전 상태를 저장했다가 필요 시 복원하는 패턴 |
| 책임 연쇄(Chain of Responsibility) 패턴 | 요청을 처리할 수 있는 객체가 순차적으로 처리하는 패턴 |
| 방문자(Visitor) 패턴 | 객체 구조는 변경하지 않고 새로운 기능을 추가하는 패턴 |

| 메시지 버스(ESB) | 연결 구조 : 버스(미들웨어) 경유<br>특징 : 미들웨어를 통해 시스템을 연결하는 방식으로 확장성과 유연성이 뛰어나고 대용량 처리에 적합 |
| 하이브리드(Hybrid) | 연결 구조 : 그룹 내 허브 + 그룹 간 버스<br>특징 : 허브 앤 스포크와 ESB를 혼합한 방식으로 관리성과 확장성을 모두 고려 |

### 주요 사이버 공격 기법

| 구분 | 내용 |
| --- | --- |
| 워터링 홀(Watering Hole) 공격 | 공격 대상이 자주 방문하는 웹사이트를 감염시킨 후 방문 시 악성코드를 감염시키는 공격 |
| 스니핑(Sniffing) | 네트워크에서 전송되는 데이터를 몰래 가로채는 공격 |
| 스푸핑(Spoofing) | IP, MAC, DNS 등을 위조하여 정상 사용자처럼 속이는 공격 |
| 피싱(Phishing) | 이메일, 문자, 가짜 사이트 등으로 개인정보를 탈취하는 공격 |
| 파밍(Pharming) | 정상 사이트 주소를 입력해도 가짜 사이트로 접속되도록 유도하는 공격 |
| 스미싱(Smishing) | 문자(SMS)에 악성 링크를 포함하여 개인정보를 탈취하는 공격 |
| 랜섬웨어(Ransomware) | 파일을 암호화한 후 복호화 대가로 금전을 요구하는 악성코드 |
| DDoS(분산 서비스 거부) 공격 | 다수의 시스템에서 대량의 트래픽을 보내 서비스를 마비시키는 공격 |
| SQL Injection | SQL문을 삽입하여 데이터베이스를 불법 조회·조작하는 공격 |
| XSS(Cross Site Scripting) | 웹페이지에 악성 스크립트를 삽입하여 사용자 정보를 탈취하는 공격 |
| CSRF(Cross Site Request Forgery) | 사용자의 권한을 이용하여 의도하지 않은 요청을 수행하게 하는 공격 |
| 제로데이(Zero-Day) 공격 | 보안 패치가 나오기 전 취약점을 이용하는 공격 |
| 백도어(Backdoor) | 정상적인 인증 절차를 우회하여 시스템에 접근할 수 있도록 만든 숨겨진 통로 |
| 키로거(Key Logger) | 사용자의 키보드 입력을 기록하여 계정, 비밀번호 등을 탈취하는 공격 |

### 소프트웨어 테스트 종류
| 구분 | 내용 |
| --- | --- |
| 베타 테스트 | 하드웨어나 소프트웨어의 개발 단계에서 상용화하기 전에 실시하는 제품 검사 작업. 제품의 결함 여부, 제품으로서의 가치 등을 평가하기 위해 실시한다. 선발된 잠재 고객으로 하여금 일정 기간 무료로 사용하게 한 후에 나타난 여러 가지 오류를 수정, 보완한다. 공식적인 제품으로 발매하기 이전에 최종적으로 실시하는 검사 작업이다. |

| 회귀 테스트(Regression Test) | 오류를 제거하거나 수정한 시스템이나 시스템 컴포넌트 또는 프로그램이 오류 제거와 수정에 의해 새로이 유입된 오류가 없는지를 확인하는 일종의 반복 시험이다. |

### 다크 데이터(Dark Data)

| 내용 |
| --- |
| 빅데이터(Big Data)와 비슷하면서도 구조화돼 있지 않고, 더는 사용하지 않는 '죽은' 데이터를 의미한다. 일반적으로 정보를 수집해 저장한 이후 분석이나 특별한 목적을 위해 활용하지 않으며, 저장공간만 차지하고 이러한 이유로 심각한 보안 위험을 초래할 수 있다. |

| 내용 |

| 내용 |
| --- |

### SSO(Single Sign-On)

| 내용 |
| --- |
| 여러 개의 사이트에서 한번의 로그인으로 여러 다른 사이트들을 자동적으로 접속하여 이용하는 방법을 말한다. 일반적으로 서로 다른 시스템 및 사이트에서 각각의 사용자 정보를 관리하게 되는데, 이때 하나의 사용자 정보를 기반으로 여러 시스템을 하나의 통합 인증으로 사용하게 한다. |

| 트로이 목마(Trojan Horse) | 정상 프로그램으로 위장하여 사용자를 속이는 악성코드<br>백도어를 생성하거나 정보를 탈취<br>자기 복제 기능 없음 |
| 랜섬웨어(Ransomware) | 파일이나 시스템을 암호화한 후 복호화 대가로 금전을 요구하는 악성코드 |
| 스파이웨어(Spyware) | 사용자 동의 없이 개인정보나 사용 정보를 수집하는 악성코드 |
| 애드웨어(Adware) | 원치 않는 광고를 지속적으로 표시하는 악성코드 |
| 키로거(Key Logger) | 키보드 입력 내용을 기록하여 계정, 비밀번호 등을 탈취하는 악성코드 |
| 루트킷(Rootkit) | 관리자 권한을 획득하고 자신의 존재를 숨기는 악성코드 |
| 백도어(Backdoor) | 정상적인 인증 절차를 우회하여 시스템에 접근할 수 있도록 만든 숨겨진 통로 |
| 봇(Bot) / 좀비 PC(Zombie PC) | 감염된 PC를 원격에서 조종하여 DDoS 등의 공격에 이용 |
| 봇넷(Botnet) | 다수의 봇(좀비 PC)으로 구성된 공격 네트워크 |
| 드로퍼(Dropper) | 다른 악성코드를 설치하기 위한 악성코드 |
| 로직 폭탄(Logic Bomb) | 특정 날짜나 조건이 만족되면 동작하는 악성코드 |

### 오류 검출 / 오류 제어 방식

| 구분 | 내용 |
| --- | --- |
| EAC(Error Auto Correction) | 수신 측이 오류를 스스로 검출하고 수정하는 방식<br>재전송 없이 오류를 복구 |
| FEC(Forward Error Correction, 전진 오류 수정) | 오류 정정 코드를 함께 전송하여 수신 측에서 오류를 검출·수정하는 방식<br>재전송이 필요 없음 |

| 해밍 코드(Hamming Code) | 패리티 비트를 이용하여 1비트 오류를 검출하고 수정하는 방식<br>2비트 오류는 검출 가능 |

### SYN Flooding

| 내용 |
| --- |
| TCP는 연결을 수립하기 위해 클라이언트가 서버에 SYN 패킷을 보내고 서버는 SYN-ACK 패킷으로 응답한 후, 클라이언트가 다시 ACK 패킷을 보내는 3-way-handshake 과정을 거친다. 이때 공격자는 수많은 SYN 패킷을 서버에 전송한 뒤 마지막 ACK를 고의로 보내지 않아 서버가 연결 대기 상태를 계속 유지하게 만든다. 이로 인해 서버의 연결 대기 큐가 가득 차 정상 접속 요청을 처리하지 못하게 된다. |

| 시스템을 폴더 모양의 단위로 구분하여 구성 요소 간 관계를 표현하는 UML 구조 다이어그램이다. 하나의 단위 안에는 여러 클래스나 하위 단위가 포함될 수 있으며, 단위 간에는 «import», «access», «merge» 등의 관계를 통해 의존성을 표현한다. 코드의 실제 폴더 구조와 비슷하게 표현되므로 소프트웨어의 모듈화, 재사용성, 의존 관계를 시각적으로 설계할 때 사용된다. |

### 화이트박스 테스트 커버리지 종류

| 구분 | 내용 |
| --- | --- |
| 문장 커버리지 (Statement Coverage) | 모든 실행 문장을 최소 1회 이상 수행하는 테스트<br>가장 기본적인 커버리지<br>키워드 : 모든 문장 실행 |
| 분기 커버리지 (Branch Coverage) | 모든 분기(True/False)를 최소 1회 이상 수행하는 테스트<br>결정문의 모든 분기 확인<br>키워드 : True, False 모두 수행 |
| 조건 커버리지 (Condition Coverage) | 각 조건식의 결과(True/False)가 모두 1회 이상 나오도록 테스트<br>개별 조건만 검사<br>키워드 : 각 조건의 True/False |
| 결정 커버리지 (Decision Coverage) | 각 결정문의 결과(True/False)를 모두 수행하는 테스트<br>분기 커버리지와 동일한 의미로 사용<br>키워드 : 결정 결과 True/False |
| 조건/결정 커버리지 (Condition/Decision Coverage) | 조건 커버리지와 결정 커버리지를 모두 만족하는 테스트<br>조건과 결정을 함께 검사<br>키워드 : 조건 + 결정 |
| 변경 조건/결정 커버리지 (MC/DC) | 하나의 조건만 변경해도 결정 결과가 바뀌는지를 확인<br>각 조건이 결정에 독립적으로 영향을 미치는지 검사<br>항공, 국방 등 안전성이 중요한 시스템에서 사용<br>키워드 : 독립 영향, 안전성 |
| 다중 조건 커버리지 (Multiple Condition Coverage) | 모든 조건의 가능한 조합(True/False)을 모두 테스트<br>가장 강력한 조건 검사 방식<br>예) 조건 2개 → TT, TF, FT, FF<br>키워드 : 모든 조합 |
| 경로 커버리지 (Path Coverage) | 프로그램의 모든 실행 경로를 최소 1회 이상 수행<br>반복문이 많으면 경로 수가 매우 증가<br>가장 높은 수준의 커버리지<br>키워드 : 모든 실행 경로 |

### 워터링 홀 공격

| 내용 |
| --- |
| 공격자는 목표 대상이 업무 또는 관심사로 인해 자주 방문하는 합법적인 웹사이트를 사전에 파악한다. |

- 해당 사이트에 악성코드를 삽입하여 감염시켜 놓고, 피해자가 해당 사이트에 접속하는 순간 피해자의 시스템에 악성 프로그램이 자동으로 설치되도록 유도한다.
- 공격자는 불특정 다수를 노리는 것이 아니라 특정 조직이나 인물을 겨냥하며, 접속자의 IP나 환경 조건을 확인하여 목표 대상에게만 선택적으로 악성코드가 실행되도록 설계하는 경우도 있다.
- 피해자는 정상적인 사이트를 방문했을 뿐이므로 감염 사실을 인지하기 어렵다는 특징이 있다.
